# 📰 Система модерации новостей для Discord бота

## Описание
Модуль системы модерации новостей с использованием базы данных MySQL. Позволяет пользователям подавать новости на модерацию, а модераторам - одобрять или отклонять их.

## Установка

### 1. Настройка переменных окружения
Добавьте в файл `.env` следующие переменные:

```env
# Токен Discord бота
DISCORD_TOKEN=your_discord_bot_token_here

# Настройки базы данных MySQL
DB_HOST=localhost
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=your_db_name

# ID каналов для системы модерации новостей
MOD_CHANNEL_ID=123456789012345678  # Канал где модераторы рассматривают заявки
NEWS_CHANNEL_ID=123456789012345678  # Канал куда публикуются одобренные новости

# ID роли модератора (для проверки прав)
MODERATOR_ROLE_ID=123456789012345678

# Тег для уведомления при публикации новости (@everyone или @here)
EVERYONE_TAG=@everyone
```

### 2. Подключение модуля
Модуль автоматически загружается как cog. Убедитесь, что файл `news_moderation.py` находится в папке `commands/`.

## Структура базы данных

### Таблица `news_requests`
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT AUTO_INCREMENT | Primary key, ID заявки |
| author_id | BIGINT | ID автора новости |
| content | TEXT | Текст новости |
| image_url | VARCHAR(512) | URL изображения (если есть) |
| status | ENUM('pending', 'approved', 'rejected') | Статус заявки |
| moderator_id | BIGINT | ID модератора, принявшего решение |
| channel_message_id | BIGINT | ID сообщения в канале модерации |
| created_at | TIMESTAMP | Дата создания заявки |
| updated_at | TIMESTAMP | Дата последнего обновления |

## Команды

### Для пользователей

#### `/submit_news` (Slash-команда)
Открывает модальное окно для подачи новости.

#### `!submit news <текст>` (Legacy команда)
Подаёт новость на модерацию.
- **Использование**: `!submit news <текст новости>`
- **Опционально**: можно прикрепить картинку к сообщению
- **Ответ бота**: «Заявка отправлена на модерацию»

#### `!mynews`
Показывает статус ваших последних заявок (до 5).

### Для модераторов

Модераторы получают сообщение в канале модерации (`MOD_CHANNEL_ID`) с:
- Embed, содержащим текст новости, автора и картинку (если была прикреплена)
- Двумя кнопками: ✅ Одобрить и ❌ Отклонить

**Важно**: Кнопки реагируют только на пользователей с ролью `MODERATOR_ROLE_ID`.

## Логика работы

### При подаче новости:
1. Заявка сохраняется в БД со статусом `pending`
2. Пользователь получает ответ: «Заявка отправлена на модерацию»
3. В канал модерации отправляется Embed с информацией о новости
4. Под сообщением размещаются кнопки «Одобрить» и «Отклонить»

### При одобрении новости:
1. Статус в БД обновляется на `approved`
2. Сообщение в канале модерации обновляется:
   - Убираются кнопки
   - Добавляется футер: «✓ Одобрено [Ник модератора]»
   - Цвет Embed меняется на зелёный
3. Новость публикуется в канале `NEWS_CHANNEL_ID`:
   - В начале добавляется тег `@everyone` (или `@here`, если настроено)
   - Сохраняется картинка, если она была
4. Автор получает ЛС: «Ваша новость опубликована!»

### При отклонении новости:
1. Статус в БД обновляется на `rejected`
2. Сообщение в канале модерации обновляется:
   - Убираются кнопки
   - Добавляется футер: «✗ Отклонено [Ник модератора]»
   - Цвет Embed меняется на красный
3. Автор получает ЛС: «Ваша новость отклонена модератором [Ник]»

## Файлы

- `commands/news_moderation.py` - основной модуль системы
- `database/mysql_connector.py` - функции для работы с БД (добавлены новые функции)

## Функции БД

- `create_news_request(author_id, content, image_url)` - создание заявки
- `get_news_request(request_id)` - получение данных заявки
- `update_news_status(request_id, status, moderator_id)` - обновление статуса
- `update_news_channel_message_id(request_id, channel_message_id)` - сохранение ID сообщения
- `get_user_news_requests(user_id, limit)` - получение заявок пользователя
- `get_pending_news_requests()` - получение всех ожидающих заявок

## Требования

- Python 3.8+
- discord.py 2.0+
- mysql-connector-python
- python-dotenv
- aiohttp (для скачивания изображений)

3. Модератор видит сообщение в канале модерации
